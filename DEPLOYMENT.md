# Устранение проблем с деплоем

## Обновление зависимостей

Если проект не собирается на Vercel, выполните следующие шаги:

### 1. Обновите все зависимости локально

```bash
# Удалите старые зависимости
rm -rf node_modules package-lock.json

# Установите заново
npm install

# Обновите до последних версий
npm update
```

### 2. Проверьте локальную сборку

```bash
npm run build
```

Если сборка успешна локально, она должна работать и на Vercel.

### 3. Типичные ошибки и решения

#### Error: Module not found
**Решение:** Убедитесь, что все импорты используют правильные пути с `@/`

#### TypeScript errors
**Решение:** Проверьте `tsconfig.json` и убедитесь, что все типы корректны

#### Build timeout
**Решение:** Упростите контент или увеличьте лимит на Vercel

### 4. Требования к версиям

- Node.js: 18.x или выше
- npm: 9.x или выше

### 5. Проверка перед деплоем

Выполните эти команды локально:

```bash
# Установка
npm install

# Проверка типов
npx tsc --noEmit

# Сборка
npm run build

# Тест продакшен сервера
npm start
```

Если все команды выполняются успешно, деплой должен пройти без проблем!

## Полезные команды

```bash
# Очистка кеша Next.js
rm -rf .next

# Полная переустановка
rm -rf node_modules package-lock.json .next
npm install

# Анализ размера бандла
npm run build -- --profile
```

## Контрольный список перед деплоем

- [ ] `npm install` проходит без ошибок
- [ ] `npm run build` успешно создаёт продакшен сборку
- [ ] Все пути импортов корректны
- [ ] TypeScript не выдаёт критических ошибок
- [ ] В package.json указаны актуальные версии
- [ ] .gitignore исключает node_modules и .next

## Логи Vercel

Если деплой всё равно не работает:

1. Откройте вкладку "Deployments" в проекте на Vercel
2. Кликните на проваленный деплой
3. Изучите полные логи сборки
4. Скопируйте ошибку и найдите решение
